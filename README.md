# Практическая работа по теме 2.4-2.5: Рекурсивные алгоритмы  
## Вариант 2: Вычисление степени числа (power(x, n))

---

### 1. Название задания

Рекурсивная функция для вычисления степени числа `x` в степени `n`.

---

### 2. Описание работы алгоритма с синтаксисом (на Python)

Алгоритм использует принцип "разделяй и властвуй" для эффективного вычисления степени. Он основан на следующих наблюдениях:
- Любое число в нулевой степени равно 1.
- Отрицательную степень можно преобразовать в положительную через обратное значение.
- Если степень чётная — результат можно получить, возведя в квадрат половину степени.
- Если степень нечётная — умножаем основание на результат возведения в степень `n-1`.

**Используемые операторы и конструкции:**
- `if` — для проверки базовых случаев (n == 0, n < 0).
- `%` — для проверки чётности степени.
- `//` — целочисленное деление, чтобы получить половину степени.
- `*` — умножение для комбинирования частей результата.
- Рекурсивный вызов функции `power(x, ...)` — для решения подзадач.

Как работает:

Базовый случай: если n == 0, возвращаем 1.
Если n < 0, возвращаем обратное значение от power(x, -n).
Если n чётное — вычисляем half = power(x, n//2), затем возвращаем half * half.
Если n нечётное — возвращаем x * power(x, n-1).
Этот подход позволяет свести задачу к логарифмическому числу шагов.

---

### 3. Временная сложность
O(log n)

---

### 4. Объяснение
На каждом рекурсивном уровне степень n уменьшается вдвое (если чётная) или на единицу (если нечётная), но в среднем — в два раза. Это означает, что глубина рекурсии равна log₂n. На каждом уровне выполняется константное количество операций (проверка условия, умножение, деление). Поэтому общая временная сложность — O(log n).

Это значительно быстрее наивной реализации (O(n)), где степень уменьшается на 1 на каждом шаге.

### 6. Ответ на контрольный вопрос №9: Чем отличаются перестановки от сочетаний?

**Перестановки** — это упорядоченные наборы элементов. Порядок элементов важен. Например, для множества {A, B, C} перестановки: ABC, ACB, BAC, BCA, CAB, CBA — всего 3! = 6 штук.

**Сочетания** — это неупорядоченные наборы элементов. Порядок элементов не важен. Например, для множества {A, B, C} сочетания по 2 элемента: AB, AC, BC — всего C(3,2) = 3 штуки.

**Ключевое отличие**: в перестановках важен порядок, в сочетаниях — нет.
